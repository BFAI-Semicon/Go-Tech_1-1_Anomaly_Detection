# Implementation Tasks

## 欠陥箇所の可視化機能

本ドキュメントは
「欠陥箇所の可視化機能」の実装タスクを定義する。
ヘキサゴナルアーキテクチャの依存方向
（Worker → Domain → Ports ← Adapters）に沿い、
ボトムアップで段階的に実装を進める。

## タスク一覧

- [x] 1. (P) Worker層: 可視化基盤の構築
- [x] 1.1 可視化種別・値オブジェクト・例外の定義
  - 可視化種別（original / heatmap / mask / overlay）
    を表す列挙型を定義する
  - 個別アーティファクト情報（ファイル名、種別、
    元画像名、相対パス）を表す
    不変の値オブジェクトを定義する
  - 収集結果のマニフェスト
    （アーティファクト一覧、CSV一覧、画像数）を
    表す不変の値オブジェクトを定義する
  - 可視化処理固有のカスタム例外クラスを定義する
  - ユニットテスト: 列挙型の値・等価性、
    値オブジェクトの不変性
  - _Requirements_: 1.1, 1.2, 1.3, 2.3

- [x] 1.2 可視化設定のYAML解析機能
  - config.yaml の visualization セクションから
    有効/無効フラグと対象種別を読み取る
  - デフォルト値（enabled=True、全種別有効）を
    適用する
  - セクション欠落時はデフォルト設定を返す
  - 不正な種別値に対してログ警告し、
    有効な値のみでフォールバックする
  - ユニットテスト: 正常系YAML解析、
    セクション欠落、不正値フォールバック、
    enabled=false判定、
    ファイル未存在時のデフォルト返却
  - _Requirements_: 6.1, 6.2, 6.3, 6.4

- [x] 2. (P) Ports/Adapters層:
  アーティファクトアクセス拡張
- [x] 2.1 ストレージポートへの
  アーティファクト操作インターフェース追加
  - 指定ジョブ・サブディレクトリ配下の
    ファイル名一覧を返す抽象メソッドを追加する。
    ディレクトリ未存在時は空リストを返す契約とする
  - 指定ジョブ・ファイルパスから
    絶対パスを返す抽象メソッドを追加する。
    パストラバーサル検出時は ValueError、
    ファイル未存在時は FileNotFoundError を送出する
  - 既存の抽象メソッドとの整合性を
    維持する（後方互換）
  - _Requirements_: 4.1, 4.2, 5.4

- [x] 2.2 ファイルシステムアダプタの
  アーティファクトアクセス実装
  - アーティファクトルートディレクトリ配下の
    ジョブ別ディレクトリをスキャンし、
    ファイル名一覧を返す実装を追加する
  - ディレクトリ未存在時に空リストを返す
    処理を実装する
  - ファイル絶対パスの解決と
    パストラバーサル防止
    （".." 含有・絶対パス拒否）を実装する
  - アーティファクトルートの環境変数
    （ARTIFACT_ROOT）対応を実装する
  - ユニットテスト: ファイル一覧（正常系）、
    空ディレクトリ、ディレクトリ未存在、
    パストラバーサル拒否、ファイル未存在の
    FileNotFoundError
  - 既存テストの後方互換性を確認する
  - _Requirements_: 4.1, 4.2, 5.4

- [x] 3. Worker層: 可視化アーティファクト収集
- [x] 3.1 可視化アーティファクトの
  収集・整理ロジック
  - 出力ディレクトリ内のPNG画像ファイルを
    再帰的に検出する
  - ファイル名パターン
    （`*_original.png`, `*_heatmap.png`,
    `*_mask.png`, `*_overlay.png`）に基づき
    種別を分類する
  - anomalib のデフォルト出力構造
    （images/ サブディレクトリ）にも対応する
  - 元画像（original）が anomalib 出力に
    含まれない場合、テストデータセットから
    対応ファイルをコピーして補完する
  - original は設定の types に関わらず
    常に収集対象とする（比較ビューの基準画像）
  - 検出したファイルを標準ディレクトリ構造
    （visualizations/ 配下）にコピー配置する。
    既に標準構造の場合は再配置をスキップする
  - CSV予測ファイル
    （image_predictions.csv,
    pixel_predictions.csv）の存在を確認し、
    マニフェストに記録する
  - 収集結果のマニフェストを生成して返却する
  - ユニットテスト: ファイル検出（再帰スキャン）、
    種別分類、元画像補完、
    標準構造への配置、CSV検出、
    マニフェスト生成、空ディレクトリ、
    設定による種別フィルタ
  - _Requirements_: 1.1, 1.2, 1.3, 1.4,
    2.1, 2.3, 2.4, 5.1, 5.2

- [x] 3.2 ジョブ実行フローへの
  可視化収集ステップ統合
  - サブプロセス完了後・メトリクス記録前に
    可視化収集ステップを挿入する
  - 可視化設定を読み取り、無効時は
    収集をスキップする
  - 収集中にエラーが発生した場合は
    WARNING レベルでログに記録し、
    メトリクス記録処理を中断せず継続する
    （グレースフルデグラデーション）
  - 既存の output_dir 全体の MLflow 記録は
    変更せず維持する（visualizations/ も
    含めて一括記録される）
  - ユニットテスト: 収集成功ケース、
    収集エラー時のメトリクス記録継続、
    可視化無効設定時のスキップ、
    サブプロセス失敗時のスキップ
  - _Requirements_: 1.5, 2.2, 5.3

- [x] 4. Domain/API層:
  可視化アーティファクト取得API
- [x] 4.1 (P) 可視化アーティファクト一覧取得の
  ドメインユースケース
  - ジョブ完了を確認した上で、
    visualizations/ 配下の画像ファイル一覧と
    ルート直下の CSV ファイル一覧を
    取得して返却するユースケースを実装する
  - 各画像ファイルに対して種別分類と
    取得用 URL を生成する
  - ジョブ未完了時は空の結果を返す
  - ユニットテスト: 正常系（画像+CSV）、
    アーティファクト不在時の空リスト、
    ジョブ未完了時の空結果
  - _Requirements_: 4.1, 4.3

- [x] 4.2 (P) 可視化 REST エンドポイントの実装
  - ジョブ別の可視化アーティファクト一覧を
    JSON で返すエンドポイントを実装する。
    レスポンスに画像ファイル一覧と
    CSV ファイル一覧を含める
  - 個別ファイル（画像・CSV）を
    FileResponse で返すエンドポイントを
    実装する。Content-Disposition ヘッダを設定する
  - 全エンドポイントに Bearer token 認証を
    適用する（既存の認証機構を再利用）
  - ジョブ ID 未存在時に 404 を返す
  - ファイル未存在時に 404 を返す
  - パストラバーサル検出時に 400 を返す
  - ユニットテスト: 認証必須確認、
    正常系 JSON レスポンス、
    FileResponse、404 応答、400 応答
  - _Requirements_: 4.1, 4.2, 4.3, 4.4, 5.4

- [x] 5. Streamlit層: 可視化結果の表示
- [x] 5.1 可視化パネル UI の実装
  - 完了済みジョブの結果詳細内に
    可視化セクション（折りたたみ式）を追加する
  - API 経由でアーティファクト一覧を取得し、
    サムネイルグリッドを描画する
  - 対象画像を選択するセレクトボックスを
    提供し、選択時に拡大表示する
  - 元画像・ヒートマップ・マスク・
    オーバーレイの 4 列比較レイアウトを
    実装する
  - MLflow UI のアーティファクトページへの
    リンクを表示する
    （既存のリンク生成ロジックを拡張）
  - アーティファクト不在時に
    「可視化結果なし」メッセージを表示する
  - _Requirements_: 3.1, 3.2, 3.3, 3.4, 3.5

- [x] 6. 統合テスト・E2E テスト
- [x] 6.1 可視化機能の統合テスト
  - ジョブ実行→可視化収集→API 取得の
    一連フローの統合テストを実装する
  - 可視化無効設定でのジョブ実行テスト
    （アーティファクト生成なし確認）を実装する
  - API エンドポイントの認証・
    レスポンス形式・404 ハンドリング・
    FileResponse の統合テストを実装する
  - パストラバーサル攻撃の防止テストを
    実装する
  - _Requirements_: 1.1-1.5, 2.1-2.4,
    4.1-4.4, 5.1-5.4, 6.1-6.4

- [ ] 6.2 (P) 可視化 UI の E2E テスト (optional)
  - 可視化パネル表示の正常系テスト
    （サムネイル・比較レイアウト）を実装する
  - アーティファクト不在時の
    UI ハンドリングテスト
    （「可視化結果なし」表示）を実装する
  - _Requirements_: 3.1-3.5

```text
Phase 2 (並列):
  Task 3 (収集ロジック, 依存: 1)
  ||
  Task 4 (Domain/API, 依存: 2)

Phase 3:
  Task 5 (Streamlit UI, 依存: 4)

Phase 4:
  Task 6 (統合テスト, 依存: 3, 4, 5)
```
